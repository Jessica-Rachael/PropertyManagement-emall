5. TRIGGER - LOG TRANSACTION STATUS UPDATE
================================================================================

TRIGGER NAME: log_transaction_status_update

FIRED ON: UPDATE on transaction table

PURPOSE:
  Automatically logs all status changes to transaction_log table
  Creates audit trail for compliance and tracking

LOGIC:
  1. Check if transaction status changed (OLD.status != NEW.status)
  2. If changed, insert record into transaction_log
  3. Records: transaction_id, old_status, new_status, timestamp, reason

LOG TABLE COLUMNS:
  • log_id - Unique log entry ID
  • transaction_id - Reference to transaction
  • old_status - Previous status (Paid/Pending)
  • new_status - New status (Paid/Pending)
  • change_timestamp - When change occurred
  • change_reason - Description of change

TESTING:
  UPDATE transaction SET status = 'Paid' WHERE transaction_id = 4;
  SELECT * FROM transaction_log WHERE transaction_id = 4;

FILE: database/views_triggers_sprocs.sql

6. STORED PROCEDURE - INSERT MONTHLY RENT
================================================================================

PROCEDURE NAME: insert_monthly_rent()

PURPOSE:
  Automatically creates pending rent transactions for all active agreements
  Ideal for monthly automation (run on 1st of each month)

PROCESS:
  1. Declare variables and cursor
  2. Loop through all active agreements
  3. For each agreement, INSERT rent transaction
  4. Set status to 'Pending'
  5. Add auto-generated notes with month/year

INPUT PARAMETERS: None (uses current date)

OUTPUT:
  Creates transaction records with:
  • Type: 'Rent'
  • Amount: monthly_rent from agreement
  • Date: current date (CURDATE())
  • Status: 'Pending'
  • Notes: "Auto-generated rent for MM/YYYY"

CALLING:
  CALL insert_monthly_rent();

EXAMPLE RESULT:
  8 new pending rent transactions inserted (one per active agreement)

FILE: database/views_triggers_sprocs.sql

TROUBLESHOOTING:

Problem: "Connection refused"
Solution: 
  1. Check MySQL running
  2. Windows: net start MySQL80
  3. Mac: brew services start mysql
  4. Linux: sudo systemctl start mysql

Problem: "Unknown database 'rental_management_system'"
Solution:
  1. Verify schema.sql executed
  2. Run: mysql -u root -p < database\schema.sql
  3. Verify: mysql -u root -p -e "SHOW DATABASES;"

Problem: "GUI won't start"
Solution:
  1. Check Python version: python --version
  2. Verify tkinter: python -m tkinter
  3. Check imports: from tkinter import *

Problem: "Foreign key constraint failed"
Solution:
  1. Ensure sample_data.sql executed
  2. Add parent records first (landlords before buildings)
  3. Use existing IDs from previous tables

Problem: "Access denied for user 'root'"
Solution:
  1. Verify MySQL password
  2. Edit database.py line 11: password = 'actual_password'
  3. Or use: mysql -u root@localhost -p
