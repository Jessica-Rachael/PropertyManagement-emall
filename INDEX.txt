  ✓ Indexes


🟢 database\sample_data.sql
──────────────────────────────────
What: Inserts 37+ sample records
When: Run SECOND
How: mysql -u root -p rental_management_system < database\sample_data.sql
Data:
  • 5 landlords (Rajesh, Priya, Amit, Sneha, Vikram)
  • 5 buildings (locations: Delhi, Bangalore, Mumbai)

Features:

  ✓ Valid references
  ✓ Mixed statuses


🟡 database\views_triggers_sprocs.sql
──────────────────────────────────
What: Creates view, trigger, stored procedure
When: Run THIRD
How: mysql -u root -p rental_management_system < database\views_triggers_sprocs.sql

1. VIEW: active_rentals
   Shows: Current active rental agreements
   Columns: Tenant, Building, Room, Rent, Days Remaining
   Query: SELECT * FROM active_rentals;

2. TRIGGER: log_transaction_status_update
   Fires: When transaction status changes
   Logs: Old status → New status + timestamp
   Table: transaction_log
   Auto: No manual action needed

3. PROCEDURE: insert_monthly_rent()
   Purpose: Auto-generate monthly rent transactions
   Call: CALL insert_monthly_rent();
   Effect: Creates pending rent for all active agreements


🟣 database\acid_demo.sql
──────────────────────────────────
What: Demonstrates ACID properties
When: Run ANYTIME (after setup)
How: mysql -u root -p rental_management_system < database\acid_demo.sql

Tests:
  1. Atomicity - Deposit transfer (all-or-nothing)
  2. Consistency - FK constraint violation
  3. Isolation - Concurrent transactions
  4. Durability - Data persistence after commit
  
Output: Test results shown in SELECT statements

COMMON OPERATIONS
════════════════════════════════════════════════════════════════════════════════

🎯 VIEW ALL LANDLORDS
  GUI: Click "View Landlords" in sidebar
  SQL: SELECT * FROM landlord;

🎯 SEARCH FOR TENANT
  GUI: Click "Search Tenant", type name, click "Filter"
  SQL: SELECT * FROM tenant WHERE CONCAT(first_name, ' ', last_name) LIKE '%name%';

🎯 CREATE NEW RENTAL AGREEMENT
  GUI: Click "Create Agreement", select tenant & room, enter dates/amounts, submit
  SQL: INSERT INTO rental_agreement (...) VALUES (...);

🎯 VIEW ACTIVE RENTALS
  GUI: Click "Active Rentals" - shows days remaining until lease expiration
  SQL: SELECT * FROM active_rentals;

🎯 RECORD PAYMENT
  GUI: Click "Record Payment", select agreement, enter amount/date, submit
  SQL: INSERT INTO transaction (...) VALUES (...);

🎯 FILTER TRANSACTIONS
  GUI: Click "Filter by Status", select "Paid" or "Pending", click "Filter"
  SQL: SELECT * FROM transaction WHERE status = 'Paid';

TROUBLESHOOTING
════════════════════════════════════════════════════════════════════════════════

❌ "Connection refused"
   ✓ Start MySQL: net start MySQL80 (Windows)
   ✓ Or use MySQL UI to start service

❌ "Unknown database"
   ✓ Run schema.sql first
   ✓ Check: mysql -u root -p -e "SHOW DATABASES;"

❌ "GUI won't start"
   ✓ Check Python: python --version
   ✓ Check tkinter: python -m tkinter
   ✓ Look at console error

❌ "Foreign key constraint error"
   ✓ Run sample_data.sql
   ✓ Or insert parent records first

❌ "Access denied"
   ✓ Check MySQL password
   ✓ Edit database.py line 11 if password exists
   ✓ Or use: mysql -u root@localhost -p
