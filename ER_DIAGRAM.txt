SUMMARY:
========
- Landlord (1) ──→ (N) Building (One landlord owns multiple buildings)
- Building (1) ──→ (N) Room (One building has multiple rooms)
- Tenant (1) ──→ (N) Rental_Agreement (One tenant can have multiple agreements)
- Room (1) ──→ (N) Rental_Agreement (One room can have multiple agreements over time)
- Rental_Agreement (1) ──→ (N) Transaction (One agreement has multiple payments)


ALL CONSTRAINTS:
================
✓ Primary Keys (PK) on all tables
✓ Foreign Keys (FK) for referential integrity
✓ NOT NULL constraints on mandatory fields
✓ ENUM types for status/category fields
✓ AUTO_INCREMENT for all primary keys
✓ Data types optimized (INT, VARCHAR, DATE, DECIMAL)
